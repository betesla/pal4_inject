# PAL4 内建手柄支持（V1）

这份文档说明 `PAL4_inject` 当前内建的手柄支持范围、默认键位和限制。

## 近期记录

- `2026-04-25`：`A/B/X/Y` 与 `Start/LB/RB/LT/RT` 已恢复到较稳定的旧版短按/分页链路；当前主要待修问题只剩左摇杆长按仍不稳定。

## 支持范围

- 输入来源：仅支持 `XInput`
- 目标设备：Xbox 手柄和大多数兼容 `XInput` 的手柄
- 不依赖 `Steam Input`
- 右摇杆、震动、原生模拟镜头暂不支持

## 启用方式

- 当前版本默认启用内建手柄支持
- 启动后会自动检测 `0` 号 `XInput` 设备
- 注入控制面板“概览”页会显示：
  - 是否启用
  - 是否已连接
  - 当前输入上下文

## 输入上下文

V1 会在三种上下文之间切换：

- `场景`
  - 普通游戏场景、迷宫探索、移动和交互
- `系统界面`
  - 通过手柄 `Start` 进入后的系统菜单
- `菜单`
  - 其他保守识别到的菜单状态

当前策略偏保守：
- `Start` 打开系统界面后，会进入“系统界面”上下文
- 再按一次 `Start` 或 `B` 会退出系统界面
- 若不是通过手柄打开的系统界面，V1 不保证所有分页状态都能被准确推断

## 默认键位

### 场景操作

- 左摇杆：
  - 映射为离散 `W/A/S/D`
  - 上下负责前进/后退
  - 左右负责转向
- `A`：
  - `SPACE`
  - 用于对话、调查、跳跃、攀爬等主确认动作
- `B`：
  - `Esc`
- `X`：
  - 按住时等价于按住鼠标左键
  - 可用于持续前进、调查等高频鼠标左键逻辑
- `Y`：
  - `R`
  - 切换走 / 跑 / 快跑
- `L3`：
  - `F`
  - 自动前进
- `Back`：
  - `M`
  - 小地图开关

### 菜单与系统界面

- 十字键：
  - 方向键导航
- `A`：
  - 确认
- `B`：
  - 返回 / 关闭
- `Start`：
  - 打开或关闭系统界面
- `LB / RB`：
  - 切换系统界面横向主分页
  - 对应 `F1 ~ F7`
- `LT / RT`：
  - 切换系统界面纵向子分页
  - 对应数字键 `1 / 2 / 3 / 4 / 5 / 6`

## 当前实现细节

- 左摇杆带 deadzone，避免轻微漂移
- 场景里的左摇杆按住状态会走 PAL4 原生 `InputManager` 键态更新链，避免输入法弹出 `a/d` 预编辑、但游戏场景实际没收到移动键的问题
- `A/B/Y/L3/Back` 这类短按键会继续按“确认 / 返回 / 切换”语义单独处理，不和左摇杆长按链混在一起
- 十字键、`LB/RB`、`LT/RT` 支持按住重复触发
- `X` 在场景里会维持鼠标左键按下状态，松开时自动释放
- 进入系统界面时，会自动释放当前已按住的场景移动键

## 已知限制

- V1 不支持右摇杆镜头
- V1 不支持模拟量移动速度
- V1 不支持自定义键位 UI
- `LT/RT` 默认按 `1~6` 六档循环切换；若当前页面并不支持数字键切换，游戏会直接忽略
- 如果某些特殊界面不是通过手柄 `Start` 打开的，`LB/RB` 与 `LT/RT` 的上下文判断可能不如系统界面稳定
