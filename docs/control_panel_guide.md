# PAL4 注入控制面板使用说明

这份文档面向发布包使用者，介绍启动游戏后可通过 `Ctrl+J` 打开/关闭的“注入控制面板”里每一项的作用。

## 面板是什么
- 这是注入器创建的独立中文控制窗口，不依赖游戏自身的 CEGUI。
- 默认隐藏，不会在启动游戏后自动弹出干扰玩家。
- 按 `Ctrl+J` 可随时显示 / 隐藏。
- 点击窗口右上角关闭按钮时，只会隐藏面板，不会关闭游戏。
- 右下角的“关闭注入”按钮才是请求本次注入退出。

## 面板底部信息
- `Ctrl+J 显示/隐藏`
  - 提示当前热键。
- `配置=...`
  - 当前配置文件实际保存路径。
  - 现在默认保存在游戏目录下的 `pal4_inject\inject_panel_settings.ini`。
- `崩溃捕获=已开启/未开启`
  - 当前 crash handler 是否已经就绪。
- `build=...`
  - 当前运行的注入构建号，主要用于排查“你跑的是不是最新包”。

## 通用列说明
除“概览”页外，其它页签中的每一行基本都包含以下几列：

- `功能`
  - 这一条 hook 或修复项的中文名称。
- `启用`
  - 是否让这一条从“仅观察”进入工作状态。
  - 关闭后会回到 `仅观察`。
  - 开启后会恢复该项记住的工作模式。
- `日志`
  - 是否把这一项的详细运行信息写入 `pal4_inject\pal4_inject_runtime.log`。
  - 默认全部关闭，避免普通玩家使用时刷太多日志。
- `模式`
  - `仅观察`
    - 只统计调用，不改游戏行为。
  - `镜像比对`
    - 为实验和对照预留，当前主要用于调试。
  - `替换（可回退）`
    - 优先走注入器实现，必要时仍可退回原逻辑。
    - 这是当前最常用、最推荐的工作模式。
  - `强制替换`
    - 更激进的替换模式，主要给实验场景预留。
- `运行状态`
  - 显示该项是否已安装、是否生效、调用次数，以及最近错误。

## 概览

### 主状态
- `引导`
  - bootstrap 是否已经完成。
- `Hook`
  - hook 系统是否已经准备好。
- `通信`
  - 注入器内部 IPC 通信是否可用。
- `崩溃捕获`
  - crash handler 是否就绪。

### 脚本模式
- `请求`
  - 启动器本次要求游戏使用 `CS` 还是 `CSB`。
- `实际`
  - 当前游戏进程实际读取到的是哪种模式。
- `flag`
  - 对应的底层标志值。
- `环境错误`
  - 如果启动参数或环境变量异常，会在这里提示。

## 输入与界面

### 界面事件替换
- 作用：
  - 接管 `ProcessUIEvent`，是当前菜单自动化、点击、填字、按键模拟最核心的一条 seam。
- 影响：
  - 键盘按下/抬起、字符输入、鼠标移动、鼠标按键、滚轮等 UI 相关输入。
- 适合：
  - 需要稳定驱动主菜单和游戏内 CEGUI 界面时开启。

### 菜单消息派发
- 作用：
  - 接管 `HandleUIMessageAndProcess`，让注入器更稳定地通过游戏自己的消息链驱动菜单。
- 影响：
  - 与 `界面事件替换` 配合，提升点击和输入的稳定性。

### 键盘兼容层
- 作用：
  - hook `SimulateKeyPressAndRelease`，把某些键盘模拟转换成更兼容的按下/抬起序列。
- 影响：
  - 主要用于兼容菜单、返回键、Esc 等交互。

### 输入帧观察
- 作用：
  - 观察每帧输入处理。
- 额外效果：
  - 会顺带刷新一部分宽屏 HUD 布局修正。
- 说明：
  - 更偏底层观察与辅助刷新，不是直接面向玩家的“开关特效”。

### 输入设备观察
- 作用：
  - 观察底层输入设备状态更新。
- 说明：
  - 偏调试用途。

### DirectInput 观察
- 作用：
  - 观察 DirectInput 初始化过程。
- 说明：
  - 偏调试用途。

### 面板焦点保护
- 作用：
  - 防止点击注入面板、再切回游戏时，PAL4 窗口被错误最小化。
- 建议：
  - 一般保持开启。

### 玩家输入观察
- 作用：
  - 预留 hook。
- 当前状态：
  - 目前不是主功能项，更多是为后续扩展保留入口。

## 脚本与文本

### 对白文本注入
- 作用：
  - hook 实际吃对白文本参数的 `giTalk` 回调。
- 默认状态：
  - 默认关闭，如需验证文本注入链路，请手动开启。
- 当前行为：
  - 主要用于验证文本注入链路。
  - 现在会把显示文本改为测试内容，同时保留语音 key，避免把语音资源路径改坏。
- 适合：
  - 验证脚本回调是否已被注入器接管。

### 动态字体重同步
- 作用：
  - 在动态 UI 字体加载完成后，按当前分辨率和宽屏布局重新同步字体缩放信息。
- 解决的问题：
  - 宽屏下部分文字发虚、缩放不对、原生分辨率不匹配。
- 建议：
  - 宽屏使用时通常建议开启。

## 渲染与画面

### 抗锯齿 MSAA
- 位置：
  - 本页签顶部单独一块，不在普通 hook 列表里。
- 可选项：
  - `关 / 2x / 4x / 8x`
- 作用：
  - 记录希望使用的 MSAA 等级，并在 D3D9 设备创建或 reset 时应用。
- 说明：
  - 修改后有时要等下一次 D3D9 reset 或重启游戏才完全体现。

### 宽屏界面居中
- 作用：
  - 让原本 4:3 设计的 UI 在宽屏下按高度等比缩放，并左右留边后居中显示，而不是直接横向拉伸。
- 解决的问题：
  - 宽屏菜单、界面元素被拉宽变形。
- 建议：
  - 宽屏分辨率下建议开启。

### CEGUI 初始化观察
- 作用：
  - 观察 CEGUI 初始化链路。
- 说明：
  - 更偏调试与定位问题。

### 小地图宽屏修正
- 作用：
  - 重新计算小地图纹理的位置和尺寸，让它和左下角 HUD 保持一致。
- 解决的问题：
  - 宽屏时小地图内容还停留在旧的 4:3 逻辑位置附近。

### 战斗浮字与胜利图居中
- 作用：
  - 修正战斗内伤害数字、状态图标、胜利图等程序控制元素的位置。
- 解决的问题：
  - 宽屏下战斗浮字、图标、胜利图错位。

### 战斗提示窗居中 A
- 作用：
  - 修正第一类战斗提示窗的位置。

### 战斗提示窗居中 B
- 作用：
  - 修正第二类战斗提示窗的位置。

### MSAA 覆写
- 作用：
  - 对应底层 D3D9 PresentParameters hook。
  - 它决定上面选择的 MSAA 值是否真正写进设备参数。
- 说明：
  - 如果这一项关闭，即使顶部 MSAA 选了 `2x/4x/8x`，实际也可能不会生效。

## 相机

### 相机俯仰保护
- 作用：
  - 在相机矩阵更新前，对俯仰角做安全夹紧。
- 解决的问题：
  - 视角越界、翻面、俯仰异常。
- 建议：
  - 一般保持开启。

## 推荐使用方式

### 普通玩家
- 保持默认配置即可。
- 当前默认策略是：
  - 渲染与画面相关项目默认启用。
  - 日志默认全部关闭。
  - 面板默认隐藏，按 `Ctrl+J` 再打开。

### 需要排查问题时
- 只打开与你要排查的那一项相关的 `日志`。
- 复现问题后查看：
  - `pal4_inject\pal4_inject_runtime.log`
- 如果游戏崩溃，再查看：
  - `pal4_inject\pal4_inject_crash_*.txt`
  - `pal4_inject\pal4_inject_crash_*.dmp`

## 相关文件
- 配置文件：
  - `pal4_inject\inject_panel_settings.ini`
- 运行日志：
  - `pal4_inject\pal4_inject_runtime.log`
- 崩溃报告：
  - `pal4_inject\pal4_inject_crash_*.txt`
- 崩溃转储：
  - `pal4_inject\pal4_inject_crash_*.dmp`

## 补充说明
- 面板中的一些项目名字看起来像“观察”，这类项目通常偏调试用途，不一定会直接改变游戏画面或玩法。
- 某些项目即使勾选了，也需要对应代码路径真正跑到时，状态里的“调用次数”才会上升。
- 如果某项显示“未安装”或持续报错，优先把日志打开后复现一次，再查看日志文件定位。
